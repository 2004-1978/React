Definición

¿Qué es React?

📦

Librería JavaScript de UI

Solo se ocupa de construir interfaces de usuario. No incluye router, ni estado global, ni HTTP — tú eliges esas herramientas.

⚡

Virtual DOM

Mantiene una copia del DOM en memoria. Calcula qué cambió y solo actualiza esa parte — no toda la página.

🧩

Basado en Componentes

La UI se divide en piezas reutilizables llamadas componentes. Cada uno maneja su propia lógica y apariencia.

🔄

Flujo Unidireccional

Los datos viajan siempre de padre a hijo (props). Hace el estado predecible y fácil de rastrear.

Fundamentos

Los 4 Pilares de React

📝

JSX

Sintaxis que mezcla HTML con JavaScript. Babel lo convierte a React.createElement() automáticamente.

📩

Props

Datos que el padre envía al hijo. Son de solo lectura — el hijo no puede modificarlos.

💾

State · useState

Memoria interna del componente. Cuando cambia con setState, React re-renderiza automáticamente.

🔁

useEffect

Ejecuta código después del render: llamadas API, suscripciones, timers. Puede reaccionar a cambios de estado.

🌐

Context API

Comparte datos entre componentes sin pasar props por cada nivel del árbol.

🔑

Keys en Listas

Identificador único que React usa para saber qué ítems de una lista cambiaron, se agregaron o borraron.

JSX en detalle

JSX —
HTML con superpoderes

JSX parece HTML pero se ejecuta como JavaScript. El compilador transforma cada etiqueta en una llamada a función.

className ≠ class onClick camelCase { } para JS Auto-cierre />
componente.jsx
// ✅ JSX — esto ES JavaScript
function Saludo({ nombre }) {
  const hora = new Date().getHours();
  const msg = hora < 12
    ? 'Buenos días'
    : 'Buenas tardes';

  return (
    <div className="card">
      {/* Expresión JS entre llaves */}
      <h1>{msg}, {nombre}!</h1>
      <button onClick={() => alert('Hola')}>
        Saludar
      </button>
    </div>
  );
}
Ejercicio Práctico

Contador
Interactivo

Componente funcional completo que demuestra estado, eventos y renderizado condicional en menos de 25 líneas.

① useState(0) — estado inicial en cero. React escucha cambios.
② Ternario JSX — renderizado condicional sin directivas.
③ c => c + 1 — updater function: forma segura de actualizar.
④ disabled={cuenta===0} — prop booleana para deshabilitar.
Contador.jsx
import { useState } from 'react';

function Contador() {

  // ① Estado local
  const [cuenta, setCuenta] = useState(0);

  // ② Renderizado condicional
  const msg = cuenta === 0
    ? 'Empieza a contar 👇'
    : `Llevas ${cuenta} clicks`;

  return (
    <div>
      <h2>{msg}</h2>
      <span>{cuenta}</span>

      {/* ③ Updater function */}
      <button onClick={() => setCuenta(c => c+1)}>
        + Incrementar
      </button>

      {/* ④ Prop booleana */}
      <button onClick={() => setCuenta(0)}
              disabled={cuenta === 0}>
        Resetear
      </button>
    </div>
  );
}

export default Contador;
Taller

App de Tareas
en 5 Pasos

Proyecto completo donde cada paso introduce un concepto nuevo sobre el anterior.

1

Crear proyecto con Vite

npm create vite@latest — ambiente moderno listo en 30s.

2

Definir estado inicial

useState con un array de objetos {id, texto, hecha}.

3

Renderizar con .map()

Cada tarea → un <li> con key única obligatoria.

4

Agregar y eliminar

Spread [...tareas, nueva] y .filter() — nunca mutar el estado.

5

Persistir con useEffect

Guardar en localStorage cada vez que tareas cambia.

App.jsx — lógica completa
const [tareas, setTareas] = useState([]);
const [input, setInput]   = useState('');

// Cargar al montar
useEffect(() => {
  setTareas(JSON.parse(
    localStorage.getItem('tareas') ?? '[]'
  ));
}, []);

// Guardar cuando cambia
useEffect(() => {
  localStorage.setItem('tareas',
    JSON.stringify(tareas));
}, [tareas]);

// Agregar — spread, nunca mutar
const agregar = () => {
  if (!input.trim()) return;
  setTareas([...tareas,
    { id: Date.now(), texto: input, hecha: false }
  ]);
  setInput('');
};

// Eliminar — filter
const eliminar = (id) =>
  setTareas(tareas.filter(t => t.id !== id));

Beneficios de React

01

Reutilización

Un componente se crea una vez y se usa en toda la app.

02

Virtual DOM

Actualiza solo lo que cambió — interfaces más rápidas.

03

React Native

Mismo conocimiento → apps iOS y Android nativas.

04

Demanda laboral

Aparece en +60% de ofertas frontend en todo el mundo.

05

Ecosistema

Next.js, Redux, React Query, Zustand y miles más.

06

Declarativo

Describes QUÉ mostrar, no CÓMO — código más legible.

07

DevTools

Extensión de navegador para inspeccionar estado en tiempo real.

08

Server Components

React 18+ permite renderizar en servidor con cero JS al cliente.
